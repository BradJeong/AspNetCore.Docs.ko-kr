<a name="how-do-i--cache-an-aspnet-page-based-upon-information-in-the-http-header"></a>[어떻게 할까요?]  HTTP 헤더에 대 한 정보를 기반으로 한 ASP.NET 페이지 캐시
====================
으로 [Chris Pels](https://twitter.com/chrispels)

이 비디오 Chris Pels 페이지의 HTTP 헤더의 정보에 따라 ASP.NET 출력 캐시의 페이지를 유지 하는 방법을 보여 줍니다. 첫째, 잠재적인 HTTP 헤더 값을 검토 합니다. 그런 다음 샘플 페이지 만들어지고 OutputCache 지시문 "허용-언어"에서 사용자의 브라우저의 언어에 따라 캐싱을 제어 하는 HTTP 헤더 값을 포함 하는 VaryByHeader 특성 사용 됩니다. 영어로 설정 된 IE에서를 선택한 다음 프랑스어를 사용 하도록 설정 되어 있는 FireFox에서 샘플 페이지가 표시 됩니다. 마지막으로, 캐시 정의 web.config 파일에서 CacheProfile를 이동 하는 옵션이 설명 되어 있습니다.
