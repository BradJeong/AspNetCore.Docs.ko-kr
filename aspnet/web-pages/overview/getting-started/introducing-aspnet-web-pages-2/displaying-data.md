이 자습서와 함께 마친 경우이 페이지 처럼 보이는 동영상 목록을 볼 수 있습니다.

## <a name="a-very-brief-introduction-to-databases"></a>데이터베이스에 대 한 매우 짧은 소개

이 자습서는 데이터베이스는 briefest 소개만 제공 합니다. 데이터베이스 경험이 있다면이 짧은 섹션을 건너뛸 수 있습니다.

데이터베이스 정보를 포함 하는 하나 이상의 테이블이 포함 &mdash; 예를 들어 테이블에 고객, 주문 및 공급 업체에 대 한 또는 학생, 교사, 클래스 및 등급에 대 한 합니다. 구조적으로 데이터베이스 테이블 스프레드시트 비슷합니다. 일반적인 주소록을 가정해 보세요. 주소록에 각 항목에 대 한 (즉, 각 사용자에 대 한) 일부의 이름, 성, 주소, 전자 메일 주소 및 전화 번호와 같은 정보를 해야 합니다.

![간단한 모눈으로 예제 데이터베이스 테이블](displaying-data/_static/image2.png)

(행은 라고도 *레코드*, 및 열은 라고도 *필드*.)

대부분의 데이터베이스 테이블에 대 한 테이블에 고객 번호, 계좌 번호 등과 같은 고유한 값을 포함 하는 열을 있어야 합니다. 이 값은 테이블의 라고 *기본 키*, 테이블의 각 행 식별을 사용 합니다. 예제에서는 ID 열은 앞의 예제에 표시 된 주소록에 대 한 기본 키를 사용 합니다.

데이터베이스에서 정보를 읽어 페이지에 표시 및 웹 응용 프로그램에서 작업을 수행 하는 작업 중 많은 구성 됩니다. 역시 사용자 로부터 정보를 수집 하는 데이터베이스에 추가 하거나 이미 데이터베이스에 있는 레코드를 수정 합니다. (다룰 것이 자습서 집합 과정에서 이러한 작업의 모든.)

데이터베이스 작업이 있으므로 매우 복잡할 수 있으며 전문된 지식 필요할 수 있습니다. 이 자습서 집합에 대 한 하지만 기본 개념에 게 모든 설명 작업을 진행 하면서를 이해 해야 합니다.

따라서 아직 수행 하지 않은 경우 WebMatrix를 열고 이전 자습서에서 만든 WebPagesMovies 사이트를 엽니다.

왼쪽된 창에서 클릭 된 **데이터베이스** 작업 영역입니다.

![WebMatrix 데이터베이스 작업 영역 탭](displaying-data/_static/image3.png)

데이터베이스 관련 작업을 표시 하는 리본 바뀝니다. 리본 메뉴에서 클릭 **새 데이터베이스**합니다.

!['새 데이터베이스' WebMatrix 리본 메뉴의 단추](displaying-data/_static/image4.png)

WebMatrix는 SQL Server CE 데이터베이스를 만듭니다 (한 *.sdf* 파일) 사이트와 동일한 이름을 가진 &mdash; *WebPagesMovies.sdf*합니다. (여기에서는이 수행 하지 않습니다 되지만 있습니다 이름을 바꿀 수 파일을 어디에을 있는 *.sdf* 확장 합니다.)

## <a name="creating-a-table"></a>테이블 만들기

리본 메뉴에서 클릭 **새 테이블**합니다. WebMatrix는 새 탭에 테이블 디자이너를 엽니다. (새 테이블 옵션을 사용할 수 없으면 있는지 확인 왼쪽의 트리 보기에서 새 데이터베이스를 선택 합니다.)

!['새 테이블' WebMatrix 리본 메뉴의 단추](displaying-data/_static/image5.png)

위쪽 (워터 마크가 표시 "Enter 테이블 이름")에 텍스트 상자에 "영화"를 입력 합니다.

![WebMatrix 데이터베이스 디자이너에서 테이블 이름을 입력](displaying-data/_static/image6.png)

아래 테이블 이름 정의 하는 경우 개별 열입니다. 에 대 한는 *동영상* 테이블이이 자습서에서는 몇 개의 열만을 만듭니다: *ID*, *제목*, *장르*, 및 *연도*.

에 **이름** 상자에 "ID"를 입력 합니다. 새 열에 대 한 모든 컨트롤을 활성화 여기에서 값을 입력 합니다.

탭에 **데이터 형식** 목록을 열고 **int**합니다. 이 값은 ID 열 (number)을 정수 데이터에 포함 되어 있음을 지정 합니다.

> [!NOTE]
> 않습니다 이름을 모든 여기 (훨씬) 있지만이 표의 이동 하려면 표준 Windows 키보드 제스처를 사용할 수 있습니다. 예를 들어 필드 사이 이동 수를 수 있습니다만 목록에 항목을 선택 하려면 입력을 시작 등입니다.


이전 탭에서 **Default Value** 상자 (즉, 비워두면). 탭에 **기본 키가** 확인란을 선택 하 고 선택 합니다. 이 옵션은 지시 데이터베이스 하는 *ID* 열에 개별 행을 식별 하는 데이터가 포함 됩니다. (즉, 각 행 해야 고유한 값을 해당 행을 찾는 데 사용할 수 있는 ID 열에 있습니다.)

선택 된 **Id** 옵션입니다. 이 옵션에 다음 일련 번호에 대 한 각각의 새 행을 생성 해야 자동으로 데이터베이스를 게 알립니다. (의 **Id** 도 선택한 경우에 옵션이 작동은 **기본 키가** 옵션입니다.)

다음 표에서 행을 클릭 하거나 tab 키를 눌러 현재 행을 두 번입니다. 제스처 중 하나는 현재 행을 저장 하 고 시작 합니다. 에 **Default Value** 열에 따르면 이제 **Null**합니다. (Null은 기본 값에 대 한 기본값 마치도록 해야 합니다.)

새 정의 되 면 **ID** 열 디자이너가이 그림과 같이 표시 됩니다.

![영화 테이블에 대 한 ID 열을 정의 하 고 나면 WebMatrix 데이터베이스 디자이너](displaying-data/_static/image7.png)

다음 열을 만들려면 클릭에서 상자에는 **이름** 열입니다. 열에 대 한 "제목"를 입력 한 다음 선택 **nvarchar** 에 대 한는 **데이터 형식** 값입니다. "Var" 부분의 **nvarchar** 이 열에 대 한 데이터가 필요에 따라 크기가 레코드 간에 다를 수 있습니다 하는 문자열을 될 것 이라는 데이터베이스에 알려 줍니다. ("N" 접두사를 나타내는 "national" 필드는 알파벳 이나 쓰기 시스템에 대 한 문자 데이터를 저장할 수 있는지를 나타내는-필드 유니코드 데이터를 보유 하는, 즉.)

선택 하는 경우 **nvarchar**, 필드에 대 한 최대 길이 입력할 수 있는 다른 상자가 나타납니다. 이 자습서에서 다루게 영화 제목 없음 50 자 보다 긴 된다는 가정 하에서 50을 입력 합니다.

Skip **Default Value** 선택을 취소 하 고는 **Null 허용** 옵션입니다. 데이터베이스에 제목이 없는 데이터베이스에 입력을 영화를 되기를 원하지 않습니다.

완료 하 고 다음 행으로 이동 하는 경우 디자이너는이 그림 같이 표시 됩니다.

![영화 테이블에 대 한 Title 열을 정의 하 고 나면 WebMatrix 데이터베이스 디자이너](displaying-data/_static/image8.png)

반복 길이 제외 하 고 "장르" 라는 열을 만들려면 다음이 단계 방금 30으로 설정 합니다.

"Year." 라는 다른 열 만들기 데이터 형식에 대 한 선택 **nchar** (하지 **nvarchar**) 길이 4로 설정 합니다. Year, 할지 "1995" 또는 "2010"와 같은 4 자리 숫자를 사용 하 여 다양 한 크기 열 필요 하지 않습니다.

완료 된 디자인의 모양은 다음과 같습니다.

![영화 테이블에 대해 정의 된 모든 필드가 후 WebMatrix 데이터베이스 디자이너](displaying-data/_static/image9.png)

Ctrl + S를 누르거나 클릭 하 고 **저장** 빠른 실행 도구 모음에서 단추입니다. 탭을 닫아 데이터베이스 디자이너를 닫습니다.

## <a name="adding-some-example-data"></a>일부 예제 데이터를 추가합니다.

이 자습서 시리즈의 뒷부분에 나오는 양식에서 새 동영상을 입력할 수 있는 페이지를 만듭니다. 그러나 지금은 다음 페이지에 표시할 수 있는 몇 가지 예제 데이터를 추가할 수 있습니다.

8 구성할 때까지 또는 하므로 여러 영화를 입력 합니다. (8 입력 쉽게 나중 페이징 작업할 수 있습니다. 그러나에 너무 많지는 않은지, 지금은 일부를 입력 합니다.) 실제 데이터가 중요 하지 않습니다.

이제 웹 페이지에서이 데이터를 표시 하도록 설정할 수 있습니다.

## <a name="displaying-data-in-a-page-by-using-the-webgrid-helper"></a>WebGrid 도우미를 사용 하 여 데이터 페이지에 표시

사용 하려는 데이터를 페이지에 표시 하려면는 `WebGrid` 도우미입니다. 이 도우미 디스플레이 표 또는 테이블 (행과 열)를 생성합니다. 볼 수 있듯이 수 상세 표 서식 및 기타 기능을 통해 지정 됩니다.

> [!TIP] 
> 
> **구조적된 쿼리 언어 (SQL)**
> 
> SQL은 데이터베이스의에서 데이터를 관리 하기 위한 대부분의 관계형 데이터베이스에서 사용 되는 언어입니다. 데이터를 검색 하 고, 업데이트 하 고 만들고 수정 하 고 데이터베이스 테이블에 데이터를 관리 하도록 하는 명령이 포함 됩니다. SQL는 프로그래밍 언어 (예: C#)와 다릅니다. Sql을 원하는 데이터베이스에 알립니다 며 데이터베이스의 작업 데이터를 가져오거나 작업을 수행 하는 방법을 알아보세요. 다음은 몇 가지 SQL 명령의 예로 및 용도입니다.
> 
> `Select * From Movies`
> 
> `SELECT ID, Name, Price FROM Product WHERE Price > 10.00 ORDER BY Name`
> 
> 첫 번째 `Select` 문의 모든 열을 가져옵니다 (지정 된 `*`)에서 *영화* 테이블입니다.
> 
> 두 번째 `Select` 의 레코드에서 ID, 이름 및 Price 열을 인출 하는 문을 *제품* Price 열 값은 10 개 이상의 테이블입니다. 이 명령은 이름 열의 값에 따라 알파벳 순서로 결과 반환 합니다. 가격 조건과 일치 하는 레코드가 없는, 빈 집합이 반환 합니다.
> 
> `INSERT INTO Product (Name, Description, Price) VALUES ('Croissant', 'A flaky delight', 1.99)`
> 
> 이 명령은에 새 레코드를 삽입는 *제품* 테이블을 1.99를 "Croissant", "A 잘 끊어지는 기쁨" 위해 Description 열 및 가격을 이름 열을 설정 합니다.
> 
> 숫자가 아닌 값을 지정 하는 단일 따옴표 (하지 큰따옴표, C#에서 같이)에 값 포함을 확인 합니다. 텍스트 또는 날짜 값 주위 하지만 숫자 주위 하지 이러한 따옴표를 사용 합니다.
> 
> `DELETE FROM Product WHERE ExpirationDate < '01/01/2008'`
> 
> 이 명령은에서 레코드를 삭제는 *제품* 테이블 열이 있는 만료 날짜는 2008 년 1 월 1 일 보다 이전입니다. (명령은 가정 하 고 *제품* 테이블에 이러한 열이 물론.) MM/DD/YYYY 형식으로 날짜를 여기서 입력 하지만 로캘에 사용 되는 형식으로 입력 해야 합니다.
> 
> `Insert` 및 `Delete` 명령 결과 집합을 반환 하지 않습니다. 대신, 이러한 명령에 의해 영향을 받은 레코드 수를 알려 주는 숫자를 반환 합니다.
> 
> 이러한 작업 (예: 레코드 삽입 및 삭제)의 일부에 대 한 작업을 요청 하는 프로세스는 데이터베이스에 대 한 적절 한 권한이 있습니다. 데이터베이스에 연결할 때 사용자 이름 및 암호를 제공 해야 하는 프로덕션 데이터베이스에 대 한 이유입니다.
> 
> SQL 명령의 수십 있지만 위의 명령과 같은 패턴을 따릅니다. SQL 명령을 사용 하 여 테이블을 만들고 데이터베이스, 테이블에서 레코드의 수를 계산, 가격을 계산 보다 많은 작업을 수행할 수 있습니다.
