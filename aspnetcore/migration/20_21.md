# <a name="migrate-from-aspnet-core-20-to-21"></a>ASP.NET Core 2.1을 2.0에서 마이그레이션

작성자: [Rick Anderson](https://twitter.com/RickAndMSFT)

참조 [ASP.NET Core 2.1의 새로운 기능](xref:aspnetcore-2.1) ASP.NET Core 2.1의 새로운 기능에 대 한 개요입니다.

이 문서의 내용

* 2.1 ASP.NET 코어 2.0 응용 프로그램 마이그레이션의 기본 사항에 설명 합니다.
* ASP.NET Core 웹 응용 프로그램 템플릿에 대해 변경 내용에 대 한 개요를 제공합니다.

2.1에는 변경 내용의 개요를 볼 수는 가장 빠른 방법은 무엇입니까?

* WebApp1 라는 ASP.NET 코어 2.0 웹 앱을 만듭니다.
* 소스 제어 시스템에는 WebApp1를 커밋하십시오.
* WebApp1를 삭제 하 고 같은 위치에 WebApp1 라는 ASP.NET Core 2.1 웹 앱을 만듭니다.
* 2.1 버전 변경 사항을 검토 합니다.

이 문서 ASP.NET Core 2.1 마이그레이션에 대 한 개요를 제공합니다. 전체 목록은 버전 2.1에 마이그레이션하는 데 필요한 모든 변경 내용이 없습니다. 일부 프로젝트의 프로젝트에 수정 사항과 프로젝트를 만들 때 선택한 옵션에 따라 더 많은 단계가 필요할 수 있습니다.

## <a name="rules-for-projects-targeting-the-shared-runtime"></a>공유 런타임을 대상으로 하는 프로젝트에 대 한 규칙

ASP.NET Core 다음 공유 런타임이 포함 되어 있습니다.

* [Microsoft.AspNetCore.App](xref:fundamentals/metapackage-app)
* [Microsoft.AspNetCore.All](xref:fundamentals/metapackage)

공유 런타임을 대상으로 하는 프로젝트에 대 한 규칙:

* 프로젝트를 참조 하는 `Microsoft.AspNetCore.All` 또는 `Microsoft.AspNetCore.App` 패키지 프로젝트의 SDK를 설정 해야 `Microsoft.NET.Sdk.Web` 프로젝트 파일의 맨 위쪽에 (`<Project Sdk="Microsoft.NET.Sdk.Web">`).
* 패키지 또는 전이적으로 참조 하는 프로젝트를 참조 하는 프로젝트 `Microsoft.AspNetCore.All` 또는 `Microsoft.AspNetCore.App`:
  * 프로젝트의 SDK로 설정 해야 `Microsoft.NET.Sdk.Web` 프로젝트 파일의 맨 위쪽에 (`<Project Sdk="Microsoft.NET.Sdk.Web">`).
  * 동일한 공유 런타임 패키지를 참조 해야 합니다. LibraryA 참조 하는 경우 `Microsoft.AspNetCore.App`, LibraryA 참조 하는 모든 프로젝트 참조 해야 `Microsoft.AspNetCore.App`합니다.
* 실행 파일 프로젝트 (채로 시작 되는 앱을 포함 하는 프로젝트 `dotnet run` 이나 실행 하 고 앱의 프로젝트를 테스트 하는 앱)에 대 한 버전 지정 해서는 안 `Microsoft.AspNetCore.App`합니다. SDK를 통해 암시적으로 버전 지정 `<PackageReference Include="Microsoft.AspNetCore.App" />`합니다.
* 프로젝트 참조 (프로젝트 참조 및 진입점 아닌 프로젝트 `Microsoft.AspNetCore.All` 또는 `Microsoft.AspNetCore.App`) 패키지 버전을 지정 해야 합니다.

## <a name="changes-to-authentication-code"></a>인증 코드를 변경

ASP.NET Core 2.1 제공 [ASP.NET Core Id](xref:security/authentication/identity) 로 [Razor 클래스 라이브러리](xref:razor-pages/ui-class) (RCL).

기본 2.1 Identity UI 하면 2.0 버전을 통해 중요 한 새 기능이 현재 제공 하지 않습니다. RCL 패키지와 함께 Identity를 교체 하는 것은 선택 사항입니다. 서식 파일을 대체 하는 이점을 RCL 버전을 사용 하 여 코드를 포함 하는 Identity를 생성 됩니다.

* 많은 파일이 소스 트리에 벗어납니다.
* 에 포함 된 모든 버그 수정 또는 새 Id로는 [Microsoft.AspNetCore.App metapackage](xref:fundamentals/metapackage-app)합니다. 자동으로 업데이트 된 Id를 가져올 때 `Microsoft.AspNetCore.App` 업데이트 됩니다.

특수을 만든 경우 서식 파일에 변경 내용이 생성 된 Identity 코드:

* 위의 장점 변환할 발생 하지 않을 수 있습니다는 RCL 버전입니다.
* ASP.NET Core 2.0 Id 코드를 유지할 수 있습니다, 완벽 하 게 지원 합니다.

사용 하 여 끝점을 노출 하는 identity 2.1는 `Identity` 영역입니다. 예를 들어 다음 표에서 Id 끝점이 2.1 2.0에서 변경 하는 예를 보여 줍니다.

| 2.0 URL         | 2.1 URL |
| --------------- | ------------ |
| / 계정/로그인  | / Identity/계정/로그인 |
| / 계정/로그 아웃 | / Identity/계정/로그 아웃 |
| / 계정/관리 | / Identity/계정/관리 |

Id를 사용 하는 코드가 있는 및 2.0 바꾸기는 응용 프로그램으로 고려할 Identity Url 2.1 Identity 라이브러리 필요한 Identity UI 사항이 `/Identity` 세그먼트 앞 Uri에 추가 합니다. 예를 들어 리디렉션을 설정 하는 Id 끝점이 새 처리 하는 한 가지 방법은 `/Account/Login` 를 `/Identity/Account/Login`합니다.

### <a name="update-identity-to-version-21"></a>버전 2.1 Id 업데이트
