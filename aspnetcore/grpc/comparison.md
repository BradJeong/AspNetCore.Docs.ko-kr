<span data-ttu-id="18a26-184">현재 브라우저에서 gRPC 서비스를 직접 호출 하는 것은 불가능 합니다.</span><span class="sxs-lookup"><span data-stu-id="18a26-184">It's impossible to directly call a gRPC service from a browser today.</span></span> <span data-ttu-id="18a26-185">gRPC는 HTTP/2 기능을 많이 사용 하며, 브라우저에서 gRPC 클라이언트를 지원 하기 위해 웹 요청에 필요한 제어 수준을 제공 하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="18a26-185">gRPC heavily uses HTTP/2 features and no browser provides the level of control required over web requests to support a gRPC client.</span></span> <span data-ttu-id="18a26-186">예를 들어, 브라우저는 호출자가 h t t p/2를 사용 하도록 요구 하거나 기본 HTTP/2 프레임에 대 한 액세스를 제공 하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="18a26-186">For example, browsers do not allow a caller to require that HTTP/2 be used, or provide access to underlying HTTP/2 frames.</span></span>

<span data-ttu-id="18a26-187">[grpc-웹](https://grpc.io/docs/tutorials/basic/web.html) 은 브라우저에서 제한적 grpc 지원을 제공 하는 grpc 팀의 추가 기술입니다.</span><span class="sxs-lookup"><span data-stu-id="18a26-187">[gRPC-Web](https://grpc.io/docs/tutorials/basic/web.html) is an additional technology from the gRPC team that provides limited gRPC support in the browser.</span></span> <span data-ttu-id="18a26-188">gRPC-웹은 모든 최신 브라우저를 지 원하는 JavaScript 클라이언트와 서버에서 gRPC-웹 프록시의 두 부분으로 구성 됩니다.</span><span class="sxs-lookup"><span data-stu-id="18a26-188">gRPC-Web consists of two parts: a JavaScript client that supports all modern browsers, and a gRPC-Web proxy on the server.</span></span> <span data-ttu-id="18a26-189">GRPC-웹 클라이언트는 프록시를 호출 하 고 grpc 서버에 대 한 gRPC 요청에서 프록시를 전달 합니다.</span><span class="sxs-lookup"><span data-stu-id="18a26-189">The gRPC-Web client calls the proxy and the proxy will forward on the gRPC requests to the gRPC server.</span></span>

### <a name="not-human-readable"></a><span data-ttu-id="18a26-192">사람이 읽을 수 없음</span><span class="sxs-lookup"><span data-stu-id="18a26-192">Not human readable</span></span>

<span data-ttu-id="18a26-193">HTTP API 요청은 텍스트로 전송 되며, 사람이 읽고 만들 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="18a26-193">HTTP API requests are sent as text and can be read and created by humans.</span></span>

<span data-ttu-id="18a26-194">gRPC 메시지는 기본적으로 Protobuf로 인코딩됩니다.</span><span class="sxs-lookup"><span data-stu-id="18a26-194">gRPC messages are encoded with Protobuf by default.</span></span> <span data-ttu-id="18a26-195">Protobuf는 송신 및 수신에 효율적 이지만, 이진 형식은 사람이 읽을 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="18a26-195">While Protobuf is efficient to send and receive, its binary format isn't human readable.</span></span> <span data-ttu-id="18a26-196">Protobuf를 사용 하려면 올바른 deserialize를 위해 파일에 지정 된 메시지의 인터페이스 설명이 *필요 합니다.*</span><span class="sxs-lookup"><span data-stu-id="18a26-196">Protobuf requires the message's interface description specified in the *.proto* file to properly deserialize.</span></span> <span data-ttu-id="18a26-197">네트워크에서 Protobuf 페이로드를 분석 하 고 요청을 직접 작성 하려면 추가 도구가 필요 합니다.</span><span class="sxs-lookup"><span data-stu-id="18a26-197">Additional tooling is required to analyze Protobuf payloads on the wire and to compose requests by hand.</span></span>

<span data-ttu-id="18a26-198">[서버 리플렉션](https://github.com/grpc/grpc/blob/master/doc/server-reflection.md) 및 [grpc 명령줄 도구](https://github.com/grpc/grpc/blob/master/doc/command_line_tool.md) 와 같은 기능은 이진 Protobuf 메시지를 지원 하기 위해 존재 합니다.</span><span class="sxs-lookup"><span data-stu-id="18a26-198">Features such as [server reflection](https://github.com/grpc/grpc/blob/master/doc/server-reflection.md) and the [gRPC command line tool](https://github.com/grpc/grpc/blob/master/doc/command_line_tool.md) exist to assist with binary Protobuf messages.</span></span> <span data-ttu-id="18a26-199">또한 Protobuf 메시지는 [JSON으로의 변환을](https://developers.google.com/protocol-buffers/docs/proto3#json)지원 합니다.</span><span class="sxs-lookup"><span data-stu-id="18a26-199">Also, Protobuf messages support [conversion to and from JSON](https://developers.google.com/protocol-buffers/docs/proto3#json).</span></span> <span data-ttu-id="18a26-200">기본 제공 JSON 변환은 디버그할 때 Protobuf 메시지를 사람이 읽을 수 있는 형식으로 변환 하는 효율적인 방법을 제공 합니다.</span><span class="sxs-lookup"><span data-stu-id="18a26-200">The built-in JSON conversion provides an efficient way to convert Protobuf messages to and from human readable form when debugging.</span></span>

<span data-ttu-id="18a26-202">다른 프레임 워크는 다음과 같은 시나리오에서 gRPC 보다 권장 됩니다.</span><span class="sxs-lookup"><span data-stu-id="18a26-202">Other frameworks are recommended over gRPC in the following scenarios:</span></span>
