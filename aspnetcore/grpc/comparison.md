<span data-ttu-id="7410e-184">현재, 브라우저에서 gRPC 서비스를 직접 호출할 수는 없습니다.</span><span class="sxs-lookup"><span data-stu-id="7410e-184">It's impossible to directly call a gRPC service from a browser today.</span></span> <span data-ttu-id="7410e-185">gRPC는 HTTP/2 기능을 많이 사용하며, 웹 요청에 대해 gRPC 클라이언트를 지원하는 데 필요한 수준의 제어 기능을 제공하는 브라우저가 없습니다.</span><span class="sxs-lookup"><span data-stu-id="7410e-185">gRPC heavily uses HTTP/2 features and no browser provides the level of control required over web requests to support a gRPC client.</span></span> <span data-ttu-id="7410e-186">예를 들어 브라우저에서 호출자가 HTTP/2를 사용하도록 요구하거나 기본 HTTP/2 프레임에 대한 액세스를 제공할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="7410e-186">For example, browsers do not allow a caller to require that HTTP/2 be used, or provide access to underlying HTTP/2 frames.</span></span>

<span data-ttu-id="7410e-187">[gRPC-웹](https://grpc.io/docs/tutorials/basic/web.html)은 브라우저에서 제한된 gRPC 지원을 제공하는 gRPC 팀의 추가 기술입니다.</span><span class="sxs-lookup"><span data-stu-id="7410e-187">[gRPC-Web](https://grpc.io/docs/tutorials/basic/web.html) is an additional technology from the gRPC team that provides limited gRPC support in the browser.</span></span> <span data-ttu-id="7410e-188">gRPC-Web은 모든 최신 브라우저를 지원하는 JavaScript 클라이언트와 서버의 gRPC-Web 프록시라는 두 부분으로 구성됩니다.</span><span class="sxs-lookup"><span data-stu-id="7410e-188">gRPC-Web consists of two parts: a JavaScript client that supports all modern browsers, and a gRPC-Web proxy on the server.</span></span> <span data-ttu-id="7410e-189">gRPC-Web 클라이언트는 프록시를 호출하고, 프록시는 gRPC 요청을 gRPC 서버로 전달합니다.</span><span class="sxs-lookup"><span data-stu-id="7410e-189">The gRPC-Web client calls the proxy and the proxy will forward on the gRPC requests to the gRPC server.</span></span>

<span data-ttu-id="7410e-190">gRPC-Web이 모든 gRPC 기능을 지원하지는 않습니다.</span><span class="sxs-lookup"><span data-stu-id="7410e-190">Not all of gRPC's features are supported by gRPC-Web.</span></span> <span data-ttu-id="7410e-191">클라이언트 및 양방향 스트리밍은 지원되지 않고, 서버 스트리밍은 제한적으로 지원됩니다.</span><span class="sxs-lookup"><span data-stu-id="7410e-191">Client and bi-directional streaming isn't supported, and there is limited support for server streaming.</span></span>

### <a name="not-human-readable"></a><span data-ttu-id="7410e-192">사람이 읽을 수 없음</span><span class="sxs-lookup"><span data-stu-id="7410e-192">Not human readable</span></span>

<span data-ttu-id="7410e-193">HTTP API 요청은 텍스트로 전송되며, 사람이 읽고 수행할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="7410e-193">HTTP API requests are sent as text and can be read and created by humans.</span></span>

<span data-ttu-id="7410e-194">gRPC 메시지는 기본적으로 Protobuf로 인코딩됩니다.</span><span class="sxs-lookup"><span data-stu-id="7410e-194">gRPC messages are encoded with Protobuf by default.</span></span> <span data-ttu-id="7410e-195">Protobuf는 효율적으로 주고받을 수 있지만, 해당 이진 형식을 사람이 읽을 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="7410e-195">While Protobuf is efficient to send and receive, its binary format isn't human readable.</span></span> <span data-ttu-id="7410e-196">Protobuf가 제대로 역직렬화하려면 *.proto* 파일에 메시지의 인터페이스 설명이 지정되어 있어야 합니다.</span><span class="sxs-lookup"><span data-stu-id="7410e-196">Protobuf requires the message's interface description specified in the *.proto* file to properly deserialize.</span></span> <span data-ttu-id="7410e-197">네트워크상에서 Protobuf 페이로드를 분석하고 요청을 직접 작성하려면 추가 도구가 필요합니다.</span><span class="sxs-lookup"><span data-stu-id="7410e-197">Additional tooling is required to analyze Protobuf payloads on the wire and to compose requests by hand.</span></span>

<span data-ttu-id="7410e-198">이진 Protobuf 메시지를 지원하기 위해 [서버 리플렉션](https://github.com/grpc/grpc/blob/master/doc/server-reflection.md) 및 [gRPC 명령줄 도구](https://github.com/grpc/grpc/blob/master/doc/command_line_tool.md)와 같은 기능이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="7410e-198">Features such as [server reflection](https://github.com/grpc/grpc/blob/master/doc/server-reflection.md) and the [gRPC command line tool](https://github.com/grpc/grpc/blob/master/doc/command_line_tool.md) exist to assist with binary Protobuf messages.</span></span> <span data-ttu-id="7410e-199">또한 Protobuf 메시지는 [JSON으로 변환 및 JSON에서 변환](https://developers.google.com/protocol-buffers/docs/proto3#json)을 지원합니다.</span><span class="sxs-lookup"><span data-stu-id="7410e-199">Also, Protobuf messages support [conversion to and from JSON](https://developers.google.com/protocol-buffers/docs/proto3#json).</span></span> <span data-ttu-id="7410e-200">기본 제공 JSON 변환은 디버그할 때 Protobuf 메시지를 사람이 읽을 수 있는 형식으로 변환하거나 그 반대로 변환하는 효율적인 방법을 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="7410e-200">The built-in JSON conversion provides an efficient way to convert Protobuf messages to and from human readable form when debugging.</span></span>

## <a name="alternative-framework-scenarios"></a><span data-ttu-id="7410e-201">대체 프레임워크 시나리오</span><span class="sxs-lookup"><span data-stu-id="7410e-201">Alternative framework scenarios</span></span>

<span data-ttu-id="7410e-202">다음과 같은 시나리오에서는 gRPC보다 다른 프레임워크를 사용하는 것이 좋습니다.</span><span class="sxs-lookup"><span data-stu-id="7410e-202">Other frameworks are recommended over gRPC in the following scenarios:</span></span>
